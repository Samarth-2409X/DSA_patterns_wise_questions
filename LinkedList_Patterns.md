# 🔗 Linked List Patterns (LeetCode)

This repository organizes Linked List problems into **core patterns** to help in quick recognition during interviews.
---

## 1. 🔁 Reversal Pattern
**👉 Idea:** Change `next` pointers

**Problems:**
1. Reverse Linked List  
2. Reverse Linked List II  
3. Reverse Nodes in K Group  

---

## 2. 🐢 Fast & Slow Pointer
**👉 Idea:** Detect cycle / find middle

**Problems:**
4. Linked List Cycle  
5. Find Duplicate Number  

---

## 3. 📏 Two Pointer (Gap Technique)
**👉 Idea:** Maintain distance between pointers

**Problems:**
6. Remove Nth Node From End of List  

---

## 4. 🔗 Merge Pattern
**👉 Idea:** Merge sorted lists

**Problems:**
7. Merge Two Sorted Lists  
8. Merge K Sorted Lists  

---

## 5. 🧠 Hashing / Mapping
**👉 Idea:** Store node references (old → new)

**Problems:**
9. Copy List With Random Pointer  

---

## 6. 🔄 Split + Reverse + Merge (Composite Pattern)
**👉 Idea:** Combination of multiple patterns

**Problems:**
10. Reorder List  

---

## 7. 🧩 Design Pattern (Linked List + Other DS)
**👉 Idea:** Real-world system design using LL + HashMap

**Problems:**
11. LRU Cache  
12. LFU Cache  
13. Design Circular Queue  

---

# 🧠 Simplified View (Golden Rule)

Instead of remembering all problems, remember core patterns:

1. Reverse  
2. Fast-Slow  
3. Two Pointer Gap  
4. Merge  
5. Hashing  
6. Design (Linked List + Map)
