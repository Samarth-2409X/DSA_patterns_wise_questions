# 📚 Stack Patterns (Core Concepts)

This repository organizes Stack problems into **4 core patterns** to simplify problem recognition and improve problem-solving speed.


---

## 📊 Pattern 1: Monotonic Stack ⚡ (MOST IMPORTANT)

**Idea:** Maintain stack in increasing/decreasing order.

**Use When:**

* Next Greater Element
* Next Smaller Element
* Range queries
* First bigger/smaller element on left/right

**Problems:**

1. Daily Temperatures
2. Online Stock Span
3. Largest Rectangle in Histogram

🧠 **One-line intuition:**
Maintain order → remove useless elements

---

## 🔄 Pattern 2: Simulation (Push-Pop Logic)

**Idea:** Simulate real-world process using stack.

**Use When:**

* Step-by-step rules
* Collision / elimination problems

**Problems:**

1. Baseball Game
2. Asteroid Collision
3. Car Fleet

🧠 **One-line intuition:**
“What should stay? What should be removed?”

---

## 🧵 Pattern 3: String / Expression Stack

**Idea:** Handle nested structures and parsing.

**Use When:**

* Brackets / parentheses
* Encoding / decoding
* Expression evaluation

**Problems:**

1. Valid Parentheses
2. Decode String
3. Simplify Path
4. Evaluate Reverse Polish Notation

🧠 **One-line intuition:**
Nested structure → use stack

---

## ⚙️ Pattern 4: Design / Augmented Stack

**Idea:** Stack with extra data or functionality.

**Use When:**

* Need O(1) operations
* Custom stack behavior

**Problems:**

1. Min Stack
2. Max Frequency Stack
3. Implement Stack / Queue

🧠 **One-line intuition:**
Store extra info along with value

---

# 🚀 FINAL SIMPLIFIED VIEW

👉 Just remember:

1. **Monotonic** → Next greater/smaller
2. **Simulation** → Collision / process
3. **String** → Brackets / parsing
4. **Design** → Special stack
